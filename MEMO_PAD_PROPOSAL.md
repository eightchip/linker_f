# メモ帳機能の提案

## 概要
タスク管理画面に、タスクとは独立したシンプルなメモ帳機能を追加します。

## 機能要件

### 1. 基本機能
- **メモの追加**: 新しいメモを簡単に追加
- **メモの編集**: 既存のメモを編集
- **メモの削除**: 不要なメモを削除
- **メモの一覧表示**: すべてのメモを時系列で表示
- **自動保存**: 編集内容を自動的に保存

### 2. UI設計

#### レイアウト
- **右側サイドパネル**: タスク一覧の右側に折りたたみ可能なパネルを配置
- **トグルボタン**: AppBarにメモ帳の表示/非表示を切り替えるボタンを追加
- **レスポンシブ**: 画面幅が狭い場合は全画面表示に切り替え

#### メモ帳パネル
```
┌─────────────────────────────────┐
│ 📝 メモ帳              [×] [検索] │
├─────────────────────────────────┤
│ [+ 新しいメモ]                   │
├─────────────────────────────────┤
│ 📄 メモ1                         │
│    2024/01/15 10:30             │
│    内容のプレビュー...            │
│    [編集] [削除]                 │
├─────────────────────────────────┤
│ 📄 メモ2                         │
│    2024/01/14 15:20             │
│    内容のプレビュー...            │
│    [編集] [削除]                 │
└─────────────────────────────────┘
```

### 3. データモデル

```dart
class MemoItem {
  final String id;
  final String content;
  final DateTime createdAt;
  final DateTime updatedAt;
  final List<String> tags; // オプション: 将来的な拡張用
}
```

### 4. 実装方針

#### ファイル構成
```
lib/
  models/
    memo_item.dart          # メモモデル
  viewmodels/
    memo_viewmodel.dart     # メモの状態管理
  views/
    task_screen.dart        # メモ帳パネルを追加
  widgets/
    memo_pad_panel.dart     # メモ帳パネルウィジェット
    memo_editor_dialog.dart # メモ編集ダイアログ
```

#### 主な機能
1. **Hiveで永続化**: メモデータをHiveに保存
2. **Riverpodで状態管理**: memoViewModelProviderで管理
3. **シンプルなUI**: 最小限の機能で使いやすく

### 5. ユーザー体験

#### 基本フロー
1. メモ帳パネルを開く（トグルボタンまたはショートカットキー）
2. 「+ 新しいメモ」をクリック
3. メモ内容を入力（ダイアログまたはインライン編集）
4. 自動保存（または保存ボタン）
5. メモ一覧に表示

#### ショートカットキー
- `Ctrl+M`: メモ帳パネルの表示/非表示を切り替え
- `Ctrl+Shift+M`: 新しいメモを追加

### 6. 拡張機能（将来のオプション）
- メモの検索機能
- メモのタグ付け
- メモのピン留め
- メモのエクスポート/インポート
- リッチテキスト対応

## 実装の優先順位

### Phase 1（基本機能）
1. メモモデルの作成
2. メモの追加・編集・削除
3. メモ一覧の表示
4. 右側パネルの実装

### Phase 2（UX改善）
1. 検索機能
2. ショートカットキー
3. 自動保存の改善

### Phase 3（拡張機能）
1. タグ機能
2. ピン留め機能
3. エクスポート機能

## 技術的な考慮事項

1. **パフォーマンス**: メモ数が増えても快適に動作
2. **データ整合性**: メモの削除・編集時のエラーハンドリング
3. **ローカライゼーション**: 日本語・英語対応
4. **アクセシビリティ**: キーボード操作とスクリーンリーダー対応
